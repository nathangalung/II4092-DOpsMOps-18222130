## 0. Executive Summary

The platform is ambitious and mostly well-layered (Kustomize overlays, ADR-driven tool choices, GitOps via ArgoCD v3.3.3), but it is **not shippable as-is**. Three structural problems dominate:

1. **Domain leakage.** 19 distinct places in `platform/` hard-code `use-case-crypto`, crypto symbols, or crypto-shaped APIs (tickers, order books, Coinbase message types). The boundary between `platform/` and `use-case-crypto/` is violated in operators (Flink, Spark, CNPG, ClickHouse, Velero), network policies, ingestion services (Go + Rust), and the entire `services/dashboard/`.
2. **End-to-end wiring is broken.** 10 critical wires between ingest → staging → processing → features → training → serving → BI do not actually connect. Several are silently placeholder (`REPLACE_WITH_*`, `:latest` tags, empty allowlists, missing `KafkaTopic` CRs, missing GE suites, missing dbt manifest for DataHub ingestion).
3. **Thesis KNF (non-functional) objectives are unverifiable.** 10 of the 12 KNF items from `materials/` cannot be measured with the current wiring (no Velero `Schedule`, no stream-throughput dashboard, no drift alert tied to Evidently, static drift threshold, no temporal-leakage guard, no audit-log retention path, inverted CPU alert, Qdrant excluded from p99, training uses `:latest` which breaks lineage).

This document lists every leak and wire defect, gives a per-component KEEP/FIX/REPLACE/REMOVE verdict, maps KNF gaps to concrete fixes, recommends five net-new additions, and sequences remediation into 4 phases.

## 2. Broken End-to-End Wires (MUST FIX — data will not flow)

The platform declares a pipeline ingest → Kafka → Iceberg → dbt → Feast → training → KServe → Superset/DataHub. **Ten links are broken.**

|---|---|---|---|---|

2. **Per-use-case overlay template.** Ship `use-case-template/` so a new domain (e.g. `use-case-iot`) copies a skeleton that only adds `KafkaTopic`, `FlinkDeployment`, `SparkApplication`, `dbt project`, `Feast FeatureView`, `KServe InferenceService`, `Schedule`, `Dashboard` — not operator configs.
3. **ADR-015 (new):** Document the KServe controller version decision; resolve the v0.17/v0.11 conflict permanently.
4. **Pre-commit + CI gates:** (a) leak regex, (b) `kustomize build` dry-run, (c) `conftest` OPA policy on operator namespaces, (d) Kyverno CLI on manifests, (e) `cosign verify` on images.
5. **Observability before optimization.** Do not REMOVE Chaos Mesh / KubeRay / Label Studio until you have SLO dashboards — absence should be deliberate, measured.
